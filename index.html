Impersonate REST

Atlassian Jira plugin that allows Jira admins to use the Jira REST API on behalf of another user.

Example Usage

To use the Jira REST API on behalf of another user, include the 'impersonate-user' header in your request. Here is an example using curl:

curl -u admin:admin -X POST -H "impersonate-user: username" -H "Content-Type: application/json" --data "{ \"fields\": { \"project\": { \"key\": \"TEST\" }, \"summary\": \"Issue summary here\", \"issuetype\": { \"name\": \"Bug\" } } }" http://localhost:8080/rest/api/2/issue

Error Handling

If the Jira user does not exist, the API will return a Bad Request response. For example:

{"error": "User username not found"}
